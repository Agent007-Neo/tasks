Введение
Цель документа
Настоящий документ описывает план тестирования веб-сайта OZON.ru в версии для слабовидящих пользователей перед выпуском обновления. 
Основная цель плана — определить стратегию, объем, подходы и критерии тестирования, чтобы обеспечить систематическое и полное выявление дефектов. 
Документ призван скоординировать усилия команды QA, минимизировать риски, связанные с релизом, и подтвердить, что ключевые функции сайта соответствуют бизнес-требованиям и ожиданиям пользователей по качеству.
Область применения
Данный план тестирования распространяется на важные для бизнеса модули и функции веб-сайта OZON.ru, включая:
Функциональность главной страницы сайта.
Функциональность просмотра карточки товаров.

Объект тестирования
Провести комплексное тестирование функциональных и нефункциональных характеристик версии для слабовидящих пользователей сайта OZON.ru для обеспечения удобства использования и отсутствия ошибок перед выпуском обновления в продакшен.
Цели тестирования
Планируется ручное тестирование главной страницы, карточек товаров и пользовательского интерфейса для слабовидящих пользователей.
Scope (скоуп/границы/объем) проекта/работ:
Входит в скоуп
Переход страницы сайта в режим для слабовидящих пользователей;
Отображение карточек товаров;
Отображение функциональных элементов управления покупкой в карточке товара: Цена с Озон картой, Количество бонусов, Оплатить позже, Выбрать другой вариант данного товара (например: цвет/размер), Добавить к сравнению, Купить в один клик, Добавить в корзину.

Тестируем только на Windows 11 + Google Chrome (последняя версия). 
Не входит в скоуп
Регистрация нового пользователя;
Поиск товара по каталогу;
Процесс оформления заказа (корзина, доставка, оплата);
Личный кабинет пользователя (история заказов);
Авторизацию уже зарегистрированного пользователя;
Удаление пользователя.
Не тестируем на всех остальных конфигурациях, включих другие операционные системы, мобильные устройства и браузеры.
Подход к тестированию
Типы тестирования
Используемые типы тестирования:
Функциональное
Юзабилити (нефункциональное)

Не будут использованы следующие нефункциональные виды: нагрузочное, безопасности и др.
Методы тестирования
Методы тестирования, который используется в проекте - ручное. 

Автоматическое тестирование не будет использовано.
Критерии
Приемочные критерии
Выполнены все запланированные тест-кейсы.
Все найденные критические дефекты исправлены.

Критерии начала работ
Реализован весь функционал, запланированный в релизе.
Система установлена на тестовый стенд.
Написаны все запланированные тест-кейсы.

Критерии завершения работ
Выполнены все запланированные тест-кейсы.
Все обнаруженные дефекты зарегистрированы.
